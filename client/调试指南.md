# 前端调试指南

## 问题排查步骤

### 1. 检查浏览器控制台
打开浏览器开发者工具（F12），查看Console标签页是否有错误信息。

### 2. 检查网络请求
在Network标签页中，检查API请求是否成功：
- 请求地址应该是 `http://localhost:3000/api/...`
- 状态码应该是200（成功）或401（未授权）

### 3. 清除浏览器缓存
- 按 `Ctrl + Shift + Delete` 清除缓存
- 或使用无痕模式打开 `http://localhost:3000`

### 4. 检查后端是否运行
访问 `http://localhost:8080/api` 应该返回JSON响应

### 5. 检查前端是否正确启动
- 确保在 `client` 目录下运行 `npm run dev`
- 应该看到类似 "Local: http://localhost:3000" 的输出

### 6. 常见问题

#### 问题：页面空白
**可能原因：**
- JavaScript错误（查看控制台）
- 路由配置问题
- 组件加载失败

**解决方法：**
1. 打开浏览器控制台查看错误
2. 检查 `http://localhost:3000` 是否正确（不要有中文路径）
3. 重新启动前端服务：`npm run dev`

#### 问题：API请求失败
**可能原因：**
- 后端未启动
- 跨域问题
- 代理配置问题

**解决方法：**
1. 确保后端在 `http://localhost:8080` 运行
2. 检查 `vite.config.js` 中的代理配置
3. 检查后端CORS配置

#### 问题：路由404
**可能原因：**
- 路由路径错误
- 组件未正确导入

**解决方法：**
1. 确保访问的是 `http://localhost:3000/` 而不是其他路径
2. 检查 `router/index.js` 中的路由配置

## 调试命令

```bash
# 前端
cd client
npm run dev

# 后端（如果使用Maven）
cd server
mvn spring-boot:run

# 或者使用IDE运行 BlogApplication.java
```

## 测试API

使用浏览器或Postman测试：

1. 健康检查：`http://localhost:8080/api`
2. 获取文章：`http://localhost:8080/api/articles`
3. 获取分类：`http://localhost:8080/api/categories`
