# 后端启动说明

## 问题：无法识别 mvn 命令

如果遇到 "无法将'mvn'项识别为 cmdlet" 的错误，说明Maven未安装或未配置到PATH。

## 解决方案

### 方案1：安装Maven（推荐）

1. **下载Maven**
   - 访问：https://maven.apache.org/download.cgi
   - 下载 `apache-maven-3.9.x-bin.zip`

2. **解压并配置**
   - 解压到某个目录，例如：`C:\Program Files\Apache\maven`
   - 添加环境变量：
     - 新建系统变量 `MAVEN_HOME` = `C:\Program Files\Apache\maven`
     - 在 `Path` 中添加：`%MAVEN_HOME%\bin`

3. **验证安装**
   ```powershell
   mvn -version
   ```

4. **启动项目**
   ```powershell
   cd server
   mvn clean install
   mvn spring-boot:run
   ```

### 方案2：使用IDE运行（最简单）

#### IntelliJ IDEA
1. 打开项目根目录
2. 等待IDEA自动识别Maven项目
3. 右键点击 `BlogApplication.java` -> Run 'BlogApplication'

#### Eclipse / VS Code
1. 导入Maven项目
2. 运行 `BlogApplication.java` 的main方法

### 方案3：使用Maven Wrapper（无需全局安装）

如果项目中有 `.mvn/wrapper` 目录，可以使用：
```powershell
.\mvnw.cmd clean install
.\mvnw.cmd spring-boot:run
```

## 数据库配置

确保MySQL已启动，数据库配置在 `src/main/resources/application.yml`：
- 数据库：blog_db（首次运行会自动创建）
- 地址：localhost:3306
- 用户名：root
- 密码：root

## 常见问题

1. **端口被占用**：修改 `application.yml` 中的 `server.port`
2. **数据库连接失败**：检查MySQL是否启动，用户名密码是否正确
3. **编译错误**：确保JDK版本为17或更高
